# Loop (oo)

**Run any project-human or AI-generated-with one command.**
Works with Docker, devcontainers, and your existing stack.
From AI-generated chaos to a predictable development workflow.

Loop (Launch Once, Operate Predictably) is a CLI that turns any project into a runnable, reliable development environment in seconds.

No docs. No drift. No "works on my machine".

---

## ⚡ Get running in 30 seconds

```id="msvjrh"
oo add
oo start
```

Done.

---

## 🧠 What just happened?

Loop:

* detects your project or workspace
* creates a runnable workflow
* installs what's missing
* starts your services

Whether your project was written by you, your team, or generated by AI-Loop makes it run.

## ⚔️ Loop vs alternatives

### 🐳 Loop vs Docker

**Docker:**

* You define containers manually
* Requires setup and maintenance
* Great for production parity

**Loop:**

* Detects and runs your workflow automatically
* No container knowledge required
* Optimized for local development speed

👉 Use Docker when you need containers
👉 Use Loop to run your project (with or without Docker)

---

### 📦 Loop vs Dev Containers (VS Code / GitHub)

**Dev Containers:**

* Requires predefined configuration
* Tied to specific editors/tools
* Slower startup (container build)

**Loop:**

* Works without pre-configuration
* Editor-agnostic
* Starts instantly, installs only what's needed

👉 Dev Containers = predefined environments
👉 Loop = **adaptive environments**

---

### 🤖 Loop vs AI tools (ChatGPT, Cursor, Copilot)

**AI tools:**

* Generate code and setups
* Often incomplete or inconsistent
* Don't guarantee runnable environments

**Loop:**

* Takes any generated project and makes it run
* Detects missing pieces
* Normalizes environments across machines

👉 AI creates
👉 Loop makes it **actually work**

### 🧰 Loop vs Package Managers (npm, pip, etc.)

---

## 🧱 Core concepts

* **workflow** -> runnable development workflow
* **service** -> reusable unit (web, api, worker, redis)
* **command** -> executable run instruction
* **requirements** -> what must be installed
* **environment setup** -> preparing machine + prerequisites
* **runtime** -> executing the workflow

```id="maavlu"

oo stop       # stop everything

```
